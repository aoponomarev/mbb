# Как получить API-ключ Yandex Cloud

## Шаг 1: Открыть раздел IAM (Identity and Access Management)

1. В Yandex Cloud Console перейдите в раздел **IAM** (Управление доступом)
   - Ссылка: https://console.yandex.cloud/iam
   - Или найдите в левом меню раздел **"IAM"** / **"Пользователи и сервисные аккаунты"**

## Шаг 2: Выбрать сервисный аккаунт

1. В разделе IAM перейдите на вкладку **"Сервисные аккаунты"** (Service Accounts)
2. Найдите сервисный аккаунт **`mbb`** (ID: `ajeudqscq65r5d5u7ras`)
3. **Кликните на имя сервисного аккаунта** `mbb`

## Шаг 3: Создать API-ключ

1. На странице сервисного аккаунта `mbb` найдите раздел **"Ключи"** или **"API Keys"**
2. Нажмите кнопку **"Создать API-ключ"** или **"Добавить ключ"**
3. В открывшемся окне:
   - Описание (опционально): можно указать "Для Cloud Function прокси YandexGPT"
   - Или просто оставить пустым
4. Нажмите **"Создать"**

## Шаг 4: Сохранить API-ключ

**⚠️ ВАЖНО**: API-ключ показывается **только один раз** при создании!

1. После создания ключа откроется окно с API-ключом
2. **Скопируйте ключ немедленно** — он больше не будет показан
3. Формат ключа: `AQVN...` (обычно начинается с `AQVN`)
4. Сохраните ключ в безопасном месте (например, в менеджере паролей или текстовом файле)

## Альтернативный путь (если не можете найти)

1. Перейдите по прямой ссылке:
   ```
   https://console.yandex.cloud/iam/service-accounts/ajeudqscq65r5d5u7ras
   ```
   (Замените `ajeudqscq65r5d5u7ras` на ваш реальный ID сервисного аккаунта, если отличается)

2. На странице сервисного аккаунта найдите раздел **"Ключи"** или **"API Keys"**
3. Нажмите **"Создать API-ключ"**
4. Скопируйте и сохраните ключ

## Что делать после получения ключа

После получения API-ключа:

1. Вернитесь в редактор Cloud Function
2. В разделе **"Переменные окружения"**:
   - Ключ: `YANDEX_API_KEY`
   - Значение: вставьте скопированный API-ключ
   - Нажмите **"Добавить"**

## Если ключ уже был создан ранее

Если ключ уже был создан, но вы его не сохранили:
- Нужно создать **новый** ключ
- Старые ключи нельзя посмотреть повторно

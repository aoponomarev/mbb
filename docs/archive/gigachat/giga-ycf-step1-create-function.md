# Шаг 1: Создание функции в Yandex Cloud Functions

> Первый шаг интеграции GigaChat через Yandex Cloud Functions

## Цель

Создать функцию в Yandex Cloud Functions для проксирования запросов к GigaChat API.

## Инструкция

### 1. Открыть Yandex Cloud Console

1. Перейдите на https://console.yandex.cloud
2. Войдите в свой аккаунт
3. Убедитесь, что у вас есть каталог (folder) для проекта
   - Если каталога нет — создайте его
   - Запишите Folder ID (например: `b1gv03a122le5a934cqj`)

### 2. Создать новую функцию

1. В левом меню найдите раздел **"Serverless"** или **"Cloud Functions"**
2. Нажмите **"Создать функцию"** (кнопка вверху страницы)
3. Заполните параметры функции:
   - **Имя**: `mbb-gigachat-proxy` (или другое на ваше усмотрение)
   - **Описание**: `Прокси для GigaChat API с поддержкой CORS и OAuth` (опционально)
4. Убедитесь, что выбран правильный каталог (folder)
5. Нажмите **"Создать"**

### 3. Выбрать среду выполнения

После создания функции вы попадете на страницу создания первой версии:
- **"Создайте первую версию функции"**
- **"Прежде чем перейти к созданию первой версии функции, выберите среду выполнения."**

1. В сетке карточек со средами выполнения найдите **Node.js**
2. **Кликните на карточку "Node.js"** (выберите версию 18, 20 или 22)
   - Рекомендуется Node.js 18 или новее (для поддержки встроенного `fetch`)
3. **Снимите галочку** с чекбокса **"Добавить файлы с примерами кода"** (если он отмечен)
   - Это важно, чтобы не добавлять лишний пример кода
4. Нажмите кнопку **"Продолжить"**

### 4. Подготовиться к следующему шагу

После нажатия "Продолжить" откроется редактор кода. **Пока ничего не делайте** — это будет следующий шаг.

## Что дальше?

После создания функции:
- ✅ Функция создана в Yandex Cloud
- ✅ Выбрана среда выполнения Node.js
- ✅ Готов редактор кода

**Следующий шаг**: Вставка кода прокси (будет в отдельном файле `giga-ycf-step2-insert-code.md`)

# Как получить API ключ GigaChat

## Шаг 1: Регистрация / Вход в аккаунт SberDevices

1. Перейдите на сайт разработчиков Sber: https://developers.sber.ru
2. Зарегистрируйтесь или войдите в существующий аккаунт
3. Подтвердите email (если требуется при регистрации)

## Шаг 2: Создание приложения

1. В личном кабинете SberDevices перейдите в раздел **GigaChat API**
   - Найти можно в каталоге продуктов или через поиск
2. Нажмите **"Подключить"** или **"Создать приложение"**
3. Заполните информацию о приложении:
   - Название приложения (например: "MBB Statistics")
   - Описание (опционально)
   - Выберите тарифный план
4. Подтвердите создание приложения

## Шаг 3: Получение Authorization Key

1. После создания приложения откройте его страницу
2. Найдите раздел **"Ключи доступа"** или **"API Keys"** или **"Авторизация"**
3. Нажмите **"Создать ключ"** или **"Генерировать ключ"**
4. Выберите тип ключа: **"Authorization Key (Base64)"**
5. Скопируйте сгенерированный ключ

**⚠️ ВАЖНО**: Authorization Key показывается **только один раз** при создании!

**Формат ключа**: Base64 строка (например: `Z2lnYWN...`)

## Шаг 4: Сохранение ключа

1. **Скопируйте ключ немедленно** — он больше не будет показан
2. Сохраните ключ в безопасном месте:
   - Менеджер паролей
   - Текстовый файл в защищенной папке
   - Переменные окружения прокси-сервиса (рекомендуется)

## Альтернативный путь (если не можете найти)

1. Перейдите в личный кабинет: https://developers.sber.ru
2. В верхнем меню найдите раздел **"Мои приложения"** или **"Кабинет разработчика"**
3. Найдите созданное приложение
4. Откройте приложение и найдите раздел с ключами доступа

## Что делать после получения ключа

После получения Authorization Key:

1. Сохраните ключ в переменных окружения прокси-сервиса:
   - Ключ: `GIGACHAT_AUTH_KEY`
   - Значение: ваш Authorization Key (Base64)
2. Используйте ключ для настройки прокси-функции
3. **НЕ передавайте ключ через клиентское приложение** — он должен быть только на сервере

## Если ключ уже был создан ранее

Если ключ уже был создан, но вы его не сохранили:
- Нужно создать **новый** ключ
- Старые ключи нельзя посмотреть повторно
- Можно создать несколько ключей для разных целей

## Важные моменты

- **Безопасность**: Authorization Key дает полный доступ к GigaChat API — храните его в секрете
- **Кэширование токенов**: После получения OAuth токена через Authorization Key, токен кэшируется и используется для запросов
- **Лимиты**: Проверьте тарифный план и лимиты использования API
